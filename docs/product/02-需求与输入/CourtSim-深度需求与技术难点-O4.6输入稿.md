## 一、已识别的困难点（请扩写：成因 / 风险 / 缓解）

1. **历史真实性与 LLM 幻觉**：人物、官职、年号、地理易错配；严格模式与架空模式的边界。  
2. **长上下文成本**：多角色多轮对话 token 爆炸；摘要与截断的语义损失。  
3. **RAG 与戏剧冲突**：检索片段可能「像史书不像戏」；如何约束模型不机械复述史料。  
4. **论辩结构可控性**：要求「反驳 / 赞同 / 攻讦」仍可能出现套路化；皇权「裁判」戏份不足或越权替臣工决策。  
5. **SQLite 向量规模**：全表余弦扫描的上限；何时迁移 sqlite-vss / 外置向量库。  
6. **多实例 SSE**：进程内总线无法跨节点；生产部署方案选择。  
7. **关系网络数据稀疏**：`relationships` JSON 若为空则图空心；如何从生成流程或用户编辑补边。  
8. **章回体风格漂移**：模型易现代网文化或过度文言；长度与章回套语平衡。  
9. **主视角与多 Agent 一致性**：「唯一变量」与「每轮单角色调用」的叙事统一。  
10. **国际化与敏感叙事**：历史事件的当代表述、教学场景合规（若面向教育）。

## 二、技术决策点（请扩写：备选方案对比表）

| 主题 | 当前/倾向 | 备选 | 决策维度 |
|------|-----------|------|----------|
| 向量存储 | Prisma Bytes + 全表扫描 | sqlite-vss / pgvector / 专用向量服务 | 规模、运维、成本 |
| 摘要时机 | 每轮结束 LLM 摘要 | 滑动窗口纯截断 / 关键句抽取 | 质量、延迟、费用 |
| 对话结构约束 | Prompt + dialogueMeta JSON | 结构化输出 schema / 二次裁判模型 | 可靠性、复杂度 |
| 时间轴 UX | 独立页 + 向导内嵌 | 纯向导单页 | 首访认知负担 |
| 关系图 | SVG 静态示意 | D3 / Cytoscape / Graphology | 交互、包体 |
| 主视角 | scenario.protagonistCharacterId | 独立 PlayableProfile 表 | 扩展性 |
